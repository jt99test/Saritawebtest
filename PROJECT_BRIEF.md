# PROJECT BRIEF

## Nombre del proyecto
SARITA

## Visión principal
Construir la web de astrología más visualmente impactante posible: cinematográfica, lujosa, inmersiva y emocionalmente resonante.

Debe sentirse menos como una web normal y más como un observatorio digital, una interfaz ritual y una obra cósmica interactiva.

La experiencia debe combinar:
- visuales impresionantes
- movimiento elegante
- sensibilidad editorial de alta gama
- entornos celestes interactivos
- funcionalidad real para carta natal
- una experiencia de lectura premium y personal

## Objetivo del producto
Crear una experiencia web en la que una persona pueda:
1. entrar en un entorno vivo tipo galaxia
2. introducir sus datos de nacimiento en un flujo bello y ritualizado
3. recibir una revelación visualmente espectacular de su carta natal
4. explorar sus planetas, casas, aspectos y temas centrales
5. sentir que la web es premium, mágica y profundamente personal

## Posicionamiento de marca
Esto no es una web genérica de horóscopos.
Esto no es kitsch, hortera ni “espiritual” de plantilla.
Esto es premium, cinematográfico, elegante, misterioso y emocionalmente inteligente.

El tono debe sentirse como:
- editorial de lujo
- mitología cósmica
- tecnología sagrada
- poético pero contenido
- íntimo y muy diseñado

## Dirección creativa
Las referencias visuales deben evocar:
- espacio profundo
- observatorios
- mecánica celeste
- sci-fi de bellas artes
- campañas de lujo
- geometría sagrada
- astronomía onírica
- interfaces premium en dark mode

La web debe sentirse:
- inmersiva
- viva
- atmosférica
- inteligente
- fluida
- cara
- memorable

## Idioma y localización
Todo el proyecto debe estar íntegramente en español de España.

Esto incluye:
- titulares
- botones
- formularios
- mensajes de error
- placeholders
- textos legales orientativos
- microcopy
- secciones editoriales
- estados vacíos
- loaders
- mensajes de éxito
- navegación

Reglas:
- no usar inglés en la interfaz salvo casos técnicos imprescindibles
- no usar español latinoamericano
- usar un castellano natural, elegante y actual
- evitar traducciones literales extrañas
- evitar un tono demasiado informal o demasiado pomposo

## Tono verbal
La voz de SARITA debe ser:
- elegante
- íntima
- evocadora
- clara
- poética con contención
- sofisticada
- emocionalmente inteligente

Debe sonar a marca premium española contemporánea.

Evitar:
- clichés de horóscopo barato
- frases vacías “espirituales”
- exageración melodramática
- tono adolescente
- anglicismos innecesarios
- lenguaje artificial o traducido

## Audiencia
Audiencia principal:
- personas interesadas en astrología y carta natal
- usuarios con sensibilidad estética
- personas que buscan una experiencia digital más personal y elevada
- usuarios dispuestos a pagar por experiencias o lecturas premium

Audiencia secundaria:
- personas espiritualmente curiosas
- usuarios que comparten experiencias visuales bonitas
- personas que buscan una alternativa más artística a las apps tradicionales de astrología

## No negociables
- La web debe sentirse premium y visualmente excepcional
- El movimiento debe sentirse intencional y elegante, nunca barato
- La tipografía debe sentirse refinada y editorial
- El tema oscuro debe seguir siendo legible y accesible
- La versión móvil es obligatoria
- El rendimiento importa; la belleza no puede romper la experiencia
- El flujo de lectura natal debe sentirse ceremonial y especial
- La carta tradicional debe convivir con una capa visual más cinematográfica
- Evitar todos los estereotipos horteras de astrología
- Toda la interfaz debe estar en español de España

## Filosofía de UX
Cada paso debe sentirse intencional.

El viaje del usuario debe sentirse como:
- llegada
- invitación
- iniciación
- revelación
- exploración

La interfaz nunca debe sentirse administrativa, corporativa o burocrática.
Incluso los momentos utilitarios deben sentirse cuidados y atmosféricos.

## Requisitos de la homepage
La homepage debe incluir:
- hero a pantalla completa
- fondo celeste vivo
- titular dramático
- CTA claro
- copy de apoyo premium
- secciones que introduzcan la lectura natal
- secciones que destaquen planetas, signos e interpretación
- bloques visualmente diferenciados
- transiciones elegantes entre secciones
- navegación y footer responsive

## Requisitos del flujo de nacimiento
La recogida de datos natales debe sentirse ritualizada y premium.

Debe incluir:
- campo de fecha
- campo de hora
- campo de lugar de nacimiento
- progresión fluida paso a paso o formulario elegante en una sola vista
- animación con gusto
- estado de carga/revelación que se sienta mágico

## Experiencia de carta natal
La revelación de la carta debe ser un momento central.

Más adelante debería soportar:
- rueda de carta
- planetas
- signos del zodiaco
- casas
- aspectos
- revelación animada
- paneles de interpretación

En las primeras fases, se aceptan datos ficticios si la revelación visual es excelente.

## Dirección de motion
El movimiento debe ser:
- suave
- elegante
- cinematográfico
- de tempo lento a medio
- atmosférico
- por capas
- nunca juguetón
- nunca distractor

Usar motion para:
- fades
- parallax
- pulsos de brillo
- deriva orbital
- ensamblaje de la carta
- estados hover
- coreografía de revelación

## Dirección de escena 3D / celeste
La capa 3D debe sentirse elegante y contenida.
No hacer un sistema solar infantil.
Planetas, orbes, partículas y estrellas deben sentirse simbólicos, artísticos y premium.

Posibles elementos:
- starfield
- capas de niebla o nebulosa
- líneas orbitales
- cuerpos planetarios brillantes
- deriva sutil de cámara
- puntos tipo constelación
- profundidad y atmósfera

## Tono del contenido
El copy debe ser:
- poético pero legible
- premium
- emocionalmente inteligente
- misterioso pero no vago
- místico pero no ridículo
- íntimo pero no melodramático

Evitar:
- clichés genéricos de horóscopo
- copy superficial del tipo “eres una mezcla de esto y lo otro”
- relleno pseudoespiritual
- sobreexplicar ideas simples

## Reglas de gusto visual
No producir:
- gradientes arcoíris neón
- efectos glitter
- planetas cartoon
- iconografía zodiacal hortera
- sci-fi chillón
- glassmorphism barato por todas partes
- pantallas recargadas
- exceso de densidad de texto
- texto con poco contraste
- animación sin propósito

## Reglas de rendimiento
- Lazy-load para sistemas visuales pesados cuando tenga sentido
- Mantener animaciones eficientes
- Evitar re-renders innecesarios
- Priorizar animaciones con transform y opacity
- Controlar el tamaño del bundle
- Ofrecer fallbacks para dispositivos menos potentes
- Respetar reduced motion
- No dejar que el 3D rompa la experiencia móvil

## Reglas de accesibilidad
- Mantener contraste fuerte en textos
- Respetar prefers-reduced-motion
- Garantizar accesibilidad por teclado cuando haya UI interactiva
- Usar HTML semántico cuando sea posible
- Hacer formularios claros y usables
- No esconder contenido importante solo tras animación

## Estructura esperada de carpetas
Organizar el código limpiamente.
Preferir carpetas como:
- components/layout
- components/home
- components/celestial
- components/chart
- components/form
- components/ui
- lib
- hooks
- store
- types
